# Collaborative Editing System Specification

## Core Architecture

### Operation Flow

```
┌─────────────────┐    ┌──────────────────┐    ┌─────────────────┐
│   LOCAL CHANGE  │───▶│   TRANSFORMATION │───▶│  APPLY LOCALLY  │
│    (User)       │    │   OF OPERATIONS  │    │   + BROADCAST   │
└─────────────────┘    └──────────────────┘    └─────────────────┘
         │                        │                       │
         │                        │                       ▼
         │              ┌──────────────────┐    ┌─────────────────┐
         └─────────────▶│   UNDO CACHE     │    │    BROADCAST    │
                        │   (History)      │    │    (Socket)     │
                        └──────────────────┘    └─────────────────┘
                                │                       │
                                │                       ▼
                        ┌──────────────────┐    ┌─────────────────┐
                        │  REMOTE CHANGE   │◀───│  RECEIVE REMOTE │
                        │  (Collaborator)  │    │   OPERATION     │
                        └──────────────────┘    └─────────────────┘
                                │
                                ▼
                        ┌──────────────────┐
                        │ TRANSFORM AGAINST│
                        │  CURRENT STATE   │
                        └──────────────────┘
                                │
                                ▼
                        ┌──────────────────┐    ┌─────────────────┐
                        │  DELETED NODES   │───▶│ REINSERT IF     │
                        │      CACHE       │    │   NECESSARY     │
                        └──────────────────┘    └─────────────────┘
```

## Fundamental Principles

### 1. Atomic Operations
- Every document change must be represented as an atomic operation
- Minimum size as possible 
- Must contain both oldState and newState for undo capability

### 2. Operation Origins
- Local operations: Generated by current user's actions
- Remote operations: Received from other collaborators
- Both types under go transformation before application
- Undo/redo only affects local operation history

### 3. State Management
- Maintain eventual consistency across all clients
- Transform remote operations against current local state
- Preserve deleted nodes in temporary cache for potential undo

## Operation Composition

### EasyOperation Structure
```
Operation {
  id: string (unique UUID)
  type: 'text-update' | 'node-delete' | 'node-insert' | 'attribute-change'
  nodeId: string (target node identifier)
  metadata: {
    oldState: any (previous state for undo)
    newState: any (new state after operation)
    selection: any (additional context)
  }
}
```

## Practical Example: Collaborative Editing Scenario

### Initial State
```
NODE-A: "Hello world"
NODE-B: "Sample text"
```

### Sequence of Events

1. **User A edits NODE-A**
   - Changes: "Hello world" → "Hello beautiful world"
   - Local operation created and applied
   - Operation saved to undo history
   - Operation broadcast to other users

2. **User B deletes NODE-A** (concurrently)
   - Delete operation created and applied locally by User B
   - NODE-A state saved to deleted nodes cache
   - Delete operation broadcast to other users

3. **User A receives delete operation**
   - Transform operation against current state
   - Apply transformed delete operation
   - Save NODE-A state to local deleted nodes cache

4. **User A performs UNDO**
   - Retrieve last operation from undo history
   - Create inverse operation (restore previous text)
   - Check if NODE-A exists in document
   - If not found, retrieve from deleted nodes cache and reinsert
   - Apply inverse operation to restored node
   - Result: NODE-A restored with "Hello world"

## Critical Considerations

- Undo/redo history contains only local operations
- Remote operations never enter undo history
- Cache size must be monitored and limited as possible
- Operation ordering must be preserved during transformation
- System must handle simultaneous conflicting operations
- User intention should be preserved across transformations
- undo/Redo will overwrite the current state of the node to 
  the saved state in the operation
